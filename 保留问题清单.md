## 🟠 P1 高危 - 保留问题 (3个)

### ⚠️ P1-3: 导出接口全量加载数据

**影响范围:**

- `MemberController.java` - 会员导出

**建议修复方案:**
```java
// 方案1: 使用 Apache POI SXSSF Workbook（流式写入）
SXSSFWorkbook workbook = new SXSSFWorkbook(100); // 内存保留100行
Sheet sheet = workbook.createSheet();
// 分批查询数据库，逐批写入

// 方案2: 使用阿里的 EasyExcel（更推荐）
EasyExcel.write(response.getOutputStream(), OrderVO.class)
    .registerWriteHandler(new LongestMatchColumnWidthStrategy())
    .sheet("订单数据")
    .doWrite(()-> orderService.listForExport());
```

**工作量评估:** 中等（约 2-3 天）
**优先级:** 中

### ⚠️ P1-4: init.sql 末尾有重复 INSERT

**问题描述:**
init.sql 文件末尾存在重复的 INSERT 语句，执行时会报错退出。

**影响:**
- 首次部署时可能失败
- 数据重复风险

**建议修复方案:**
```sql
-- 1. 查找重复记录
SELECT phone, COUNT(*) as cnt
FROM sys_user
GROUP BY phone
HAVING cnt > 1;

-- 2. 删除重复数据，保留 ID 最小的那条
DELETE FROM sys_user
WHERE id NOT IN (
    SELECT MIN(id) FROM sys_user GROUP BY phone
);

-- 3. 添加唯一约束防止未来重复
ALTER TABLE sys_user ADD UNIQUE INDEX idx_phone (phone);
```

**工作量评估:** 简单（1-2 小时）
**优先级:** 低（数据清理时处理）

**影响范围:**

**建议修复方案:**
```java
// 使用 BigDecimal 进行精确计算
BigDecimal overdueAmount = totalPayable
    .multiply(new BigDecimal("0.05"))  // 逾期比例
    .setScale(2, RoundingMode.HALF_UP);  // 保留2位小数

// 边界情况检查
if (dueDate.isAfter(LocalDate.now()) && !isAlreadyOverdue) {
    // 未逾期，不计算逾期费用
    overdueAmount = BigDecimal.ZERO;
}
```

### ⚠️ P2-1: Controller 业务逻辑过重

**问题描述:**
Controller 中包含过多业务逻辑，应迁移到 Service 层。

**当前状态:**
- `OrderController.java` - 约 800 行
- `FactoryController.java` - 约 600 行
- `FinanceController.java` - 约 500 行

**建议修复方案:**
```
重构前:
Controller → 直接操作 Mapper

重构后:
Controller → Service → Mapper
```

**示例 - 将订单逻辑迁移到 Service:**
```java
// OrderService.java
public class OrderService {

    @Resource
    private OrderMapper orderMapper;

    @Resource
    private RedisTemplate<String, String> redisTemplate;

    @Transactional
    public Order createOrder(CreateOrderDTO dto) {
        // 1. 生成订单编号
        String orderNo = generateOrderNo();

        // 2. 构建订单实体
        Order order = new Order();
        order.setOrderNo(orderNo);
        // ... 其他字段映射

        // 3. 保存订单
        orderMapper.insert(order);

        // 4. 保存物料明细
        saveOrderMaterials(order.getId(), dto.getMaterials());

        return order;
    }

    // ... 其他业务方法
}
```

**工作量评估:** 较大（约 1 周）
**优先级:** 中（不影响功能，但影响可维护性）

```java
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT,
    username VARCHAR(50),
    operation VARCHAR(100),
    module VARCHAR(50),
    method VARCHAR(200),
    params TEXT,
    result VARCHAR(20),
    error_msg TEXT,
    ip VARCHAR(50),
    create_time DATETIME
);

// 2. 使用 AOP 记录
@Aspect
@Component
public class OperationLogAspect {

    @Around("@annotation(operationLog)")
    public Object around(ProceedingJoinPoint point, OperationLog operationLog) {
        // 记录操作前后状态
        // 入库保存
    }
}
```

**工作量评估:** 中等（约 2-3 天）

### 💡 P3-7: 数据库表缺少外键约束

**问题描述:**

**建议修复方案:**
```sql
-- 示例：订单表关联客户表
ALTER TABLE ord_order
ADD CONSTRAINT fk_order_customer
FOREIGN KEY (customer_id) REFERENCES ord_customer(id);

-- 示例：订单物料关联订单表
ALTER TABLE ord_order_material
ADD CONSTRAINT fk_material_order
FOREIGN KEY (order_id) REFERENCES ord_order(id);
```

**注意:** 添加外键前需确保数据一致性。

**工作量评估:** 简单（1-2 天，需数据验证）
