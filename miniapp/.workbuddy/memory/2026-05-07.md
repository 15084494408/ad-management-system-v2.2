## API 对接（Spring Boot 后端）
- Auth: `POST /auth/login` → `{token, userInfo: {id, username, realName, avatar, roles}}`
- Dashboard: `GET /dashboard/stats` → 今日订单/收款/totalCustomers/stockWarnings 等
- Dashboard: `GET /dashboard/board` → KPI/kpi.unpaidAmount/unfinishedOrders
- Orders: `GET /orders?current=1&size=10`
- Orders Detail: `GET /orders/{id}` → `{order, materials, ...}`
- Orders Status: `PATCH /orders/{id}/status` → `{status: 4}`
- Orders Payment: `POST /orders/{id}/payment` → `{amount}`
- Customers: `GET /customers?current=1&size=20`
- Customer Detail: `GET /customers/{id}`
- Customer Create: `POST /customers` → `{name, phone, address, customerType, ...}`
- Customer Recharge: `POST /customers/{id}/recharge` → `{amount, remark}`
- Upgrade Member: `POST /customers/{id}/upgrade-member`
- Todo: `GET /todo/list` → `[{items: [...], statusCount}]`
- Todo Status: `PATCH /todo/{id}/status` → `{status: 4}`
- Notice: `GET /notice?current=1&size=50`

## 后端 Status 码约定
- Order: 1=待处理, 2=进行中, 3=已完成, 4=已取消
- Todo: 1=新收集, 2=分析中, 3=待确认, 4=已转订单
- Payment: 1=未付, 2=部分付, 3=已付清, 4=抹零

## 字段映射
- Customer 实体 `customerName` 经 `@JsonProperty("name")` 映射为 JSON 的 `name`
- 登录响应用 `realName` 而非 `nickname`
- 分页响应统一 `PageResult{total, current, size, records}`
- 统一响应 `Result{code, message, data}` (code=200 成功)
